# fn-15-07t Ecosystem Command Center - Cross-Project Integration Hub

## Overview

Transform xano-v2-admin from a standalone V1→V2 migration tool into the **central command center** for the entire Agent Dashboards ecosystem. Three projects share the same Xano backends but currently have zero visibility into each other. This epic adds a new "Ecosystem" tab group with cross-project links, unified record counts, test user matrix, V2 integration tracking, and sync pipeline health — all in one place.

### The Three Projects

| Project                | GitHub                                                                                            | Purpose                                             | Tech                              |
| ---------------------- | ------------------------------------------------------------------------------------------------- | --------------------------------------------------- | --------------------------------- |
| **dashboards2.0**      | [scottman1078/dashboards2.0](https://github.com/scottman1078/dashboards2.0)                       | Production BI frontend, 22+ dashboard pages         | Next.js 16, React 19, SWR, Vercel |
| **v0-demo-sync-admin** | [roboulos/v0-demo-sync-admin-interface](https://github.com/roboulos/v0-demo-sync-admin-interface) | Demo data management, 7 tabs, 188+ endpoints tested | Next.js 16, React 19, Vitest      |
| **xano-v2-admin**      | [roboulos/xano-v2-admin](https://github.com/roboulos/xano-v2-admin)                               | V1→V2 migration admin, 23 tabs, proof system        | Next.js 16, React 19, React Query |

### Shared Backend

- **V1 Workspace:** `xmpx-swi5-tlvy.n7c.xano.io` (251 tables, 25.4M+ records)
- **V2 Workspace:** `x2nu-xcjc-vhax.agentdashboards.xano.io` (193 tables)
- **Sync:** V1→V2 tracks 5 entities (887K records = 3.5% of total)

## Scope

### In Scope

- New "Ecosystem" tab category in page.tsx (3 new tabs)
- New API routes for cross-project data (`/api/ecosystem/*`)
- GitHub API integration for repo status (public repos, no auth needed)
- V1 record count visualization (25.4M records by category)
- Test user reference matrix across all three projects
- Sync pipeline health aggregation
- Cross-project documentation links

### Out of Scope

- Modifying dashboards2.0 or v0-demo-sync code (read-only integration)
- GitHub Actions or CI/CD changes
- Authentication between projects
- Real-time WebSocket connections to other projects
- Mobile layouts

## Approach

### New Tab Group: "Ecosystem" (added to page.tsx viewModes)

Add 3 new tabs after the existing Interactive Proof System group:

```
Ecosystem (3 tabs):
  - Ecosystem Hub       (Building2 icon)  → Cross-project dashboard with links, status, health
  - Record Census       (BarChart3 icon)  → V1 25.4M record breakdown + migration scope
  - Test Users          (Users icon)      → Unified test user matrix across all projects
```

### Data Sources

1. **GitHub REST API** (unauthenticated, 60 req/hr) — last commit, branch info, open PRs
2. **Existing Xano endpoints** — `/api:20LTQtIX/sync-v1-to-v2-direct` for sync status
3. **Existing Xano endpoint** — `workspace-record-counts` (ID 18650, already built) for V1 counts
4. **Static config** — project URLs, deployment links, test user data
5. **New Next.js routes** — `/api/ecosystem/*` for aggregation
