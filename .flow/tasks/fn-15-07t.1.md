# fn-15-07t.1 GitHub Status API Route - /api/ecosystem/github-status proxy

## Description

Create a Next.js API route at `/api/ecosystem/github-status/route.ts` that proxies GitHub REST API calls for all three repos. This keeps GitHub API calls server-side (avoids CORS) and allows response caching.

### Implementation

**File:** `app/api/ecosystem/github-status/route.ts`

**Repos to query:**

- `scottman1078/dashboards2.0`
- `roboulos/v0-demo-sync-admin-interface`
- `roboulos/xano-v2-admin`

**GitHub API calls per repo (unauthenticated, 60 req/hr limit):**

1. `GET /repos/{owner}/{repo}` — stars, forks, default branch, updated_at
2. `GET /repos/{owner}/{repo}/commits?per_page=1` — last commit message, author, date
3. `GET /repos/{owner}/{repo}/branches` — branch count + names

**Response shape:**

```typescript
{
  repos: {
    [repoKey: string]: {
      name: string
      full_name: string
      html_url: string
      description: string | null
      default_branch: string
      updated_at: string
      last_commit: {
        message: string
        author: string
        date: string
        sha: string
      }
      branches: string[]
      open_issues_count: number
    }
  }
  rate_limit: { remaining: number, reset: number }
  timestamp: string
}
```

**Error handling:** Use `Promise.allSettled` so one repo failure doesn't block the others. Return partial data with error flags per repo.

**Caching:** Set `Cache-Control: public, s-maxage=300` (5 minutes) on the response.

## Acceptance

- [ ] Each repo includes branch list
- [ ] Partial failures return partial data (not 500)
- [ ] Rate limit info included in response
